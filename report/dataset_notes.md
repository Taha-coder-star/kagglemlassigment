# Dataset Notes — Playground Series S6E4: Predicting Irrigation Need

## Problem Summary

Given soil properties, weather conditions, crop characteristics, and farming practices for agricultural fields in India, predict how much irrigation the field needs. This is a practical water-resource management problem synthetically generated by Kaggle for competition purposes.

---

## Dataset Shapes

| File | Rows | Columns |
|------|------|---------|
| train.csv | 630,000 | 21 (20 features + target) |
| test.csv | 270,000 | 20 (no target) |
| sample_submission.csv | 270,000 | 2 (id + Irrigation_Need) |

Total labelled examples: 630,000. Test set is ~30% of the full data.

---

## Target Column

**`Irrigation_Need`** — string label, 3 classes:

| Class | Count | Share |
|-------|-------|-------|
| Low | 369,917 | 58.7% |
| Medium | 239,074 | 38.0% |
| High | 21,009 | 3.3% |

The dataset is **imbalanced**. `High` is severely underrepresented (~3.3%). A naive model that always predicts `Low` would get high accuracy but poor macro F1. Class imbalance must be handled explicitly.

---

## Number of Classes

**3** — ordered: Low < Medium < High (ordinal relationship exists but is not enforced by default classifiers).

---

## Missing Values

**None.** Every column in both train and test is fully populated. No imputation required.

## Problem Type

**Multiclass Classification** (3 classes). The classes have a natural ordinal ordering (Low → Medium → High), but treating it as flat multiclass is the standard approach unless ordinal models show a benefit.

---

## Evaluation Metric

The sample submission uses string labels (`Low`, `Medium`, `High`). Kaggle Playground Series competitions with imbalanced multiclass targets typically use **macro-averaged F1 score** — each class is weighted equally regardless of frequency. This penalises models that ignore the rare `High` class.

Optimise for: `sklearn.metrics.f1_score(y_true, y_pred, average='macro')`
